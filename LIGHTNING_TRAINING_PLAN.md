## Model Analysis

**TinyTribeV3** (`tiny_tribe/v3_model.py`):

| Component | Params | Notes |
|---|---|---|
| Projectors (3×) | ~3.5M | text/audio/video → 512-dim |
| Temporal motion (video) | ~1.5K | depthwise Conv1D |
| Embeddings | ~2M | per-modality time + pos embeds |
| MoE Transformer (4 layers) | ~18M | 8 experts, top-2, HRF bias in layers 0-1 |
| Gate pool + HRF conv | ~3K | gated modality fusion + Gamma HRF kernel |
| Output MLP + FiLM | ~0.3M | |
| `vertex_proj` at 20484 vertices | **10.5M** | Linear(512, 20484) |
| `feat_proj` (KD alignment) | 0.6M | Linear(512, 1152) |
| **Total** | **~35M** | at n_vertices=20484 |

**Key bugs in v3_train.py (fixed in train_lightning.py):**
1. Model returns `out["prediction"]` (singular), not `out["predictions"]`
2. Model output shape is `(B, n_v, T)`, not `(B, T, n_v)` — teacher must be transposed
3. `forward()` has no `modality_dropout_p` arg — model uses `self.modality_dropout` internally when `model.train()` is active

## Training Strategy

### Phase 2 (Pure KD) — what we actually do with 200 clips

No real fMRI → pure knowledge distillation from teacher predictions.

```
student(video_feat, audio≈0, text=0, subject_id=0) → pred (B, n_v, T)
loss = KD(pred, teacher_preds)
```

**Loss breakdown:**
- `output_kd` (MSE) — 60% weight, main signal
- `temporal_coherence` (penalise TR-to-TR jitter) — 10%
- `multi_resolution` (MSE at 2× and 4× downsampled) — 5%
- `aux_loss` (MoE load balancing) — 1%
- `feature_kd` (cosine in 1152-dim space) — 20% **when teacher fusion features available** (not in our dataset → 0)

Since we don't have teacher fusion features (`fusion_l4`), effective weights are:
- `output_kd`: 0.60, `temporal`: 0.10, `multi_res`: 0.05, `aux`: 0.01

### Optimiser

- AdamW, lr=3e-4, wd=1e-2, betas=(0.9, 0.98)
- Cosine decay with 3-epoch warmup
- Gradient clipping: 1.0
- Mixed precision (bf16 on A10G, fp16 on T4)

### Hyperparameters

```
batch_size = 16
seq_len    = 5   (each clip = 1 sample, T=5)
epochs     = 100
n_vertices = 20484
n_subjects = 1   (single subject-agnostic run)
```

## Feature Extraction

Run `tiny_tribe/extract_features_v3.py` before training:
1. Load each mp4 from `distillation_dataset/`
2. Sample T=5 uniformly-spaced frames → MobileViT-S → `(5, 640)` video feat
3. Load full audio @ 16kHz → Whisper-Tiny encoder → temporal mean → `(5, 384)` audio feat (ambient noise)
4. Text = zeros `(5, 384)` — no speech
5. Load `preds.npy` as teacher `(5, 20484)`
6. Save `features/<stem>.pt` with keys: `text, audio, video, teacher, subject_id`

**Estimated extraction time:** ~2–3s per clip on CPU → ~10 min for 200 clips. Run locally.

## Commands

### Local: extract features
```bash
cd tribe
pip install timm transformers torchaudio av
python tiny_tribe/extract_features_v3.py \
    --dataset_dir ./distillation_dataset \
    --out_dir ./features
```

### Lightning AI: train
Upload `tiny_tribe/`, `features/` to Lightning Studio then:
```bash
pip install lightning torchmetrics matplotlib
python tiny_tribe/train_lightning.py \
    --features_dir ./features \
    --save_dir ./checkpoints \
    --epochs 100 \
    --batch_size 16
```

## Known Limitations

1. **200 clips is very small** — student will likely overfit on video→fMRI mapping. Use modality dropout (0.3) + weight decay to regularise.
2. **No real fMRI** — we measure how well student mimics teacher, not actual brain prediction accuracy. Pearson r here is teacher-agreement, not brain-r.
3. **Single subject** — FiLM conditioner has `n_subjects=1`. All videos treated as same subject.
4. **No speech** — text features are zeros, audio is ambient noise. Model converges purely on video features.
5. **T=5 per clip** — very short sequences. Temporal coherence loss adds minimal benefit but doesn't hurt.
